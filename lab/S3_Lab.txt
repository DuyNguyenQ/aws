# Lab2
1. Tạo 1 user có quyền S3 full access
2. Login bằng user vừa tạo ở 1 và thử thực hiện một số thao tác trên S3
3. Thay đổi S3 policy, explicit deny user ở 1 không cho access
4. Thử access lại bằng user vừa tạo ở 1
    => Kết luận S3 bucket policy sẽ combine với IAM policy khi đánh giá
    quyền của user tuy nhiên deny được ưu tiên hơn allow nên mặc dù user
    có quyền S3Full vẫn không thể truy cập nếu setting của một S3 bucket là deny
5. Clear resource: xoá user tạo ở 1
